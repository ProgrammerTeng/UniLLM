# 产品方案：多模型聚合调用平台
**版本：** v2.0 | **日期：** 2026-03-19
**定位：** 对标 Geneasy.AI + Tiandun.AI，面向企业客户的生产级 LLM API 统一调用层
**竞品测试基础：** 两大竞品深度测试（335K+ tokens），详见竞品测试报告

---

## 一、产品定位

### 一句话定位
> **"帮助 AI 应用公司和互联网企业，用一个 API 接入所有主流大模型，并提供生产级的调用治理、稳定性保障和成本管控。"**

### 目标客户优先级
| 优先级 | 客户类型 | 痛点 | 月 API 消费 |
|--------|---------|------|-----------|
| P0 | AI APP 公司（如 AI 写作、AI 客服） | 多模型切换成本高、稳定性差 | $1k-$10k |
| P0 | 垂类模型公司（RAG/微调服务商） | 需要基础 LLM + Embedding 能力 | $500-$5k |
| P1 | 中型互联网公司（ToB/ToC 产品） | 合规要求 + 成本控制 + 使用量审计 | $5k-$50k |

### 核心竞争壁垒

1. **全覆盖**：OpenAI + Claude + Gemini + 国产模型 = 唯一一个全部可用的平台

3. **Gemini 智能处理**：解决 reasoning_tokens 截断问题，用户可配置 reasoning budget
4. **企业级治理**：Fallback、路由、审计、项目隔离（竞品均无）

## 三、产品功能规划

### MVP（第 1 个月）：把基础做对

**核心功能：**
1. **统一 Chat API**（OpenAI 兼容）
   - 接入：GPT-4o/o1/o3、Claude Opus/Sonnet/Haiku 全系、Gemini 2.5/3、DeepSeek V3.2、Qwen、Doubao
   - 全面支持 streaming、function calling（含多工具并行）、vision（URL + base64）
   - `/v1/models` **仅返回真实可用模型**（区别于竞品虚标行为）
   - Gemini reasoning_tokens 透明展示 + 可配置 reasoning budget

2. **Embedding API**
   - text-embedding-3-small/large（OpenAI）
   - bge-m3、embedding-v3（阿里/智谱）

3. **用量计费面板**
   - 实时 token 消耗统计（含 reasoning_tokens 分项）
   - 按模型、按日期分维度展示
   - 余额预警通知

4. **API Key 管理**
   - 多 Key 创建、权限控制（只读 vs 完整）
   - Key 级别用量隔离

5. **模型健康 Status Page**（公开）
   - 实时显示每个模型的可用性、平均延迟、成功率
   - 历史可用性报告（日/周/月）
   - **竞品都没有，这是信任壁垒**

**技术栈（参考）：**
```
前端: Next.js + shadcn/ui
后端: Go (Gin) — 高并发场景更合适
数据库: PostgreSQL + Redis
部署: Vercel (前端) + 腾讯云/阿里云 (后端)
计费: Stripe（海外）/ 支付宝（国内）
监控: Prometheus + Grafana (Status Page 数据源)
```

### V1.1（第 2-3 个月）：企业级能力

5. **智能路由与 Fallback**
   - 主备模型配置（gpt-4o 失败 → claude-sonnet 自动切换）
   - 按延迟/成本/质量打分动态路由
   - SLA 监控 + 自动降级
   - **多工具 Function Calling 兼容层**：对不支持多工具的模型（Gemini/DeepSeek）自动拆分为多次单工具调用

6. **项目管理 & 团队协作**
   - 多项目隔离（每项目独立 Key、预算、用量）
   - 子账号 + 角色权限（Owner/Admin/Developer/Viewer）
   - 审计日志（谁在什么时候调用了什么模型）

7. **成本管控**
   - 按项目设置月度预算上限
   - 超额自动暂停 / 告警
   - 模型成本对比报告（含 reasoning 成本分析）

### V1.2（第 4-6 个月）：差异化护城河

8. **图像生成 API**
   - DALL-E 3、Flux Pro、Stable Diffusion

9. **模型评测 & 推荐引擎**
   - 内置 benchmark（基于我们的竞品测试方法论）
   - 按任务类型（代码/写作/推理/数学）推荐最优模型
   - 展示实测数据：延迟、输出质量、一致性评分

10. **合规功能**
    - 内容安全过滤（Moderation）
    - 数据不出境选项（国内节点）
    - 企业签约 + 保密协议

## 四、定价策略

### 定价逻辑
- **透传定价**：成本 + 20-40% markup（参考 OpenRouter）
- 国产模型 markup 可更高（竞品都做不到，稳定性有护城河）
- 企业客户：月消费 >$1000 给 15% 折扣 + 专属支持
- **Gemini reasoning_tokens**：仅对用户可见的 output_tokens 计费，reasoning 不额外收费 → 定价优势

### 分级套餐

| 套餐 | 价格 | 特性 |
|------|------|------|
| **开发者** | 免费 | $5 免费额度，限速 10 RPM |
| **专业版** | $29/月 | 无限额度，60 RPM，基础 Dashboard |
| **团队版** | $99/月 | 多 Key + 项目管理 + 审计日志 |
| **企业版** | 定制 | SLA + 专属支持 + 合规功能 + Fallback |

## 五、GTM（上市策略）

### 第一阶段：种子期（0-1 个月）
**目标：** 前 50 个付费用户

1. **冷启动渠道：**
   - 加入 AI 开发者群（微信群、Discord）提供免费试用
   - V2EX / 掘金 / 少数派 写竞品对比测评文章（利用我们的测试数据）
   - GitHub 开源一个 OpenAI 兼容的 SDK，带流量
   - **发布竞品测试报告**（脱敏版），建立专业形象

2. **产品钩子：**
   - 注册送 $10 免费额度
   - 开发者方案永久免费（限速）
   - 一键迁移：改一行代码（base_url）即可从 OpenAI/竞品切换
   - **公开 Status Page**：展示我们 vs 竞品的可用性差距

### 第二阶段：增长期（2-4 个月）
**目标：** MRR $10k

1. **口碑增长：**
   - 月度模型可用性报告（对标竞品，建立公信力）
   - 开放 API 用量分析功能
   - 模型推荐引擎免费使用

2. **渠道拓展：**
   - 与 AI 应用框架（LangChain、Dify、FastGPT）合作集成
   - 联系 AI APP 加速器、AI 创业社群

### 第三阶段：企业期（5-12 个月）
**目标：** MRR $50k+，签约 10+ 企业客户

1. **企业销售：**
   - 直销：针对 AI APP 公司（月消费 >$1k 的目标客户）
   - 渠道：与云计算代理商合作

## 七、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| API 供应链不稳定 | 高 | 多渠道采购同一模型；fallback 机制；参考竞品经验 — Geneasy 可用的我们优先接 |
| OpenAI 封号/限制 | 高 | 备用账号池；直接接入 Azure OpenAI |
| Claude 接入难度 | 高 | 研究 Geneasy.AI 的 Claude 接入方式（可能通过第三方渠道），准备直接 Anthropic API |
| 价格竞争 | 中 | 以模型全覆盖 + 诚信透明 + 企业功能差异化 |
| 合规/数据安全 | 中 | 明确数据不存储承诺；支持私有部署 |
| Gemini reasoning 成本 | 中 | 仅对可见输出计费，差异化定价策略 |
| 竞品跟进 | 低 | 快速迭代；企业客户关系壁垒；公开数据建立信任护城河 |
